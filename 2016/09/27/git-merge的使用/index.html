git merge的使用

1、创建分支

git checkout -b dev

创建dev分支，相当于先git branch dev，再git checkout dev。

2、在dev分支做修改后提交

修改某个文件之后运行

3、切换回主分支

git checkout master

此时回到主分支再查看文件夹的话就看不到刚才你修改或创建的文件。

4、合并分支

git merge dev

此时再查看当前文件夹的话就能看到刚才修改或创建的文件了。

5、删除分支

git branch -d dev

删除之后再用git branch命令查看就只有master分支了。

6、提交代码

首先要更新本地代码

git pull origin master

之后提交

git push origin master
